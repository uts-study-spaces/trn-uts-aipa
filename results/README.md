
The modelling pipeline removes duplicate tickets across these files and excludes the two German-normalized CSV files from the main classifier because their queue/category labels use a different taxonomy.

The selected model is Linear SVM for both queue/category prediction and priority prediction. These files can be used for the report and presentation, but they should be regenerated if the dataset, preprocessing, model settings, or evaluation split changes.

The transformer embedding benchmark is optional and sampled. It compares multilingual semantic embeddings with a simple supervised classifier, so it should be interpreted cautiously against the full-data TF-IDF results.

Streamlit reads several files from this folder:

- `metrics_summary.csv` for final model metrics and the Overview model comparison chart.
- `per_language_metrics.csv` and `per_class_f1.csv` for detailed report tables.
- `transformer_embedding_benchmark.csv` for the optional transformer comparison table and chart.
- `dataset_profile.json` for dataset size, source-file count, and deduplication figures.

The app does not use transformer embeddings for live prediction. Live single-ticket and batch predictions use the trained TF-IDF Linear SVM model files in `models/`.
